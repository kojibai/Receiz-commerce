# First Paint and Effects

Render known exact artifact/local truth immediately. Construct and call the remote domain only behind that settled proof surface. A remote artifact must be independently verified before it can append verified additions.

Named-domain acceptance and external effect delivery are distinct. The acceptance transaction records only an outbox intent. Use `resolveEffectStatus` for delivery state. Retry or dead-letter state cannot reverse accepted artifact truth.

Never let network, database, session, Connect, receipt, projection, or effect state replace the sealed proof object or delay deterministic first paint.
